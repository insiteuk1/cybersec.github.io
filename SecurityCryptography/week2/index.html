Concept of Cryptography

Introduction

What is Cryptography

Cryptography (Greek for “hidden, secret, and writing”) is the practice, evaluation, and analysis of approaches and techniques to establish security services, such as confidentiality, integrity, and authenticity within a communication environment to establish data security.

Symmetric Key Cryptography

Symmetric Key Cryptography is a cryptographic method where a unique key is shared by both the sender and receiver. This key is used for both encryption and decryption of messages, ensuring that only the intended recipient can decrypt the message with the same key.

Asymmetric Key Cryptography

Asymmetric Key Cryptography, on the other hand, involves separate keys for both the sender and receiver. One key, known as the public key, is used for encryption, while the other, the private key, is used for decryption. This method provides an added layer of security as the private key is kept secret, and only the public key is shared.

Symmetric Key Cryptography

Symmetric Key Cryptography is a fundamental cryptographic method where a single, shared key is used for both encryption and decryption of messages. The sender and receiver must possess the same key for secure communication. When a message needs to be sent securely, the sender uses this key to encrypt the plaintext into ciphertext. The ciphertext is then transmitted over an insecure channel (e.g., the internet) and can only be decrypted by the recipient using the same shared key. Symmetric key cryptography is known for its efficiency, especially when dealing with large amounts of data, but it necessitates that both parties securely share and maintain the secrecy of the key.

Cryptanalysis

Cryptanalysis is the practice and study of complex statistical and mathematical techniques used to obtain meaningful information about the ciphertext or encrypted message. It involves various methods and approaches to analyze and break cryptographic systems. In practice, it is crucial never to use an untested cryptosystem, as affirmed by Kerckhoff's Principle (1883).

Kerckhoff's Principle states that a cryptosystem should preserve its security properties even if everything about the cryptosystem, including encryption and decryption algorithms, is made public, provided the secret key is kept securely. This principle emphasizes the importance of relying on the secrecy of the key rather than the secrecy of the algorithm. It encourages the use of well-established and thoroughly tested cryptographic systems to ensure security.

Cryptosystem

A cryptosystem is a fundamental concept in cryptography, consisting of a 5-tuple represented as (E, D, M, K, C), where each component plays a crucial role in secure communication.

The components of a cryptosystem are as follows:

  • M: The set of plaintexts
  • K: The set of keys
  • C: The set of ciphertexts
  • E: The encryption algorithm, denoted as E: M × K ⇒ C
  • D: The decryption algorithm, denoted as D: C × K ⇒ M

Cryptosystems are designed to provide security services, such as confidentiality, integrity, and authenticity, within a communication environment to establish data security. Understanding the components and operations of a cryptosystem is essential for ensuring secure communication.

Two basic properties should be established in a cryptosystem:

  • Correctness Property: When a secret key, K, is used to encrypt a message, M, with an encryption algorithm producing a ciphertext, C, the decryption algorithm with the same key, K, should give back the original message, M.
  • Secrecy Property: The ciphertext, C, should not reveal any information about the key and the original message, M.

Substitution Cipher

The Substitution Cipher, also known as the monoalphabetic substitution cipher, is a classic encryption technique that operates on letters within a plaintext message. It involves a one-to-one mapping of each plaintext letter to a fixed ciphertext letter.

For example, if the letter 'A' in the plaintext is mapped to the letter 'D' in the ciphertext, every occurrence of 'A' in the message is replaced with 'D'.

The Substitution Cipher is a historical encryption method, but it raises questions about its security. The main concerns include:

  • Is the cipher secure?
  • How can we attack the cipher?

Substitution Cipher Attack

The Substitution Cipher, while seemingly secure at first glance, is susceptible to several cryptographic attacks. Here are some common attack methods:

  • Exhaustive Key Search (Brute-Force Attack): Attackers can attempt every possible substitution table (each table is a key) until an intelligible plaintext message appears. However, this approach becomes infeasible due to the vast key space.
  • Letter Frequency Analysis Attack: This attack relies on the inherent characteristics of the English language, such as the frequency of letter occurrence. For example, the letter 'E' occurs most frequently, followed by 'T' and 'A'. Attackers can exploit the preservation of letter frequency from plaintext to ciphertext to decipher the message.

Substitution Cipher (Caesar Cipher)

The Caesar cipher is one of the simplest approaches to encryption, and it operates by shifting every character in the plaintext a fixed number of positions to the right. This fixed number, denoted as 'k,' represents the key of the cipher. Let's explore how it works:

Example:
Plain Text: SECURITY AND CRYPTOGRAPHY
Ciphertext: bnldarch= 1 13 11 3 0 17 2 7

In this example, the key 'k' is 9, which means that each letter in the plaintext is shifted nine positions to the right in the alphabet. Note that the letters "wrap around" at the end of the alphabet, which can be mathematically expressed as a reduction modulo 26. For instance, 18 + 9 = 27 ≡ 1 mod 26.

Modulo Arithmetic (and a clock)

Modulo Arithmetic is a fundamental concept in cryptography, especially for computations within finite sets. It involves the use of a modulus to perform operations that wrap around when a certain value is reached. Let's explore this concept using a clock as an example.

Goal

The goal of Modulo Arithmetic is to perform computations within finite sets. To illustrate this concept, consider an example with a finite set of integers. Imagine a clock with 12 integers, which allows us to introduce the idea of modular arithmetic. In this case, the clock is modulo 12, meaning it wraps around after reaching 12.

Modular Arithmetic

In its simplest form, modulus is essentially the remainder from division. For example, if we divide 14 by 6, the result is 2 with a remainder of 2. In modular arithmetic:

  • 0 mod 3 = 0
  • 2 mod 3 = 2
  • 7 mod 7 = 0
  • 15 mod 10 = 5

In general, for two positive numbers 'a' and 'n,' 'a modulo n' (abbreviated as 𝑎 𝑚𝑜𝑑 𝑛) is the remainder of the division of 'a' by 'n,' where 'a' is the dividend and 'n' is the divisor. Modulo arithmetic is crucial for various cryptographic operations and computations within finite sets.

Caesar Cipher

The Caesar Cipher can be represented in the form of modulo expressions with keys 'k,' plaintext 'm,' and ciphertext 'c' belonging to the set {0, 1, …, 25}. The encryption and decryption processes are defined as follows:

Encryption:

c = ek(m) = (m + k) mod 26

Decryption:

m = dk(c) = (c - k) mod 26

In these expressions, 'k' represents the key, 'm' is the plaintext, and 'c' is the ciphertext. The modulo operation ensures that the result falls within the range of 0 to 25, corresponding to the 26 letters of the alphabet.

Attack on Caesar Cipher

The Caesar Cipher, despite its simplicity, is vulnerable to various attacks:

  • Brute Force Attack or Exhaustive Key Search:

    • This attack involves trying all possible 26 keys until meaningful text is revealed. Since there are only 26 possible keys, it's a straightforward attack.

  • Letter Frequency Analysis:

    • Similar to the frequency attack in substitution cipher, this method relies on the frequency distribution of letters in the English language to identify the key and decrypt the message.

Dictionary Attack

Rule:

  • First letter in a word becomes A
  • Second letter becomes B
  • Letter repeated occurrences in the plaintext use the same letter

Example:

  • HELLO → ABCCD
  • CELLS → ABCCD
  • SPEAKER → ABCDECF
  • ELEPHANT → ABACDEFG
  • FOOTBALL → ABBCDEFF

In a Dictionary Attack, the plaintext is transformed into a sequence of letters based on the rules mentioned above. For example, the word "HELLO" is transformed into "ABCCD." This transformation is applied to the entire plaintext message.

Converting Word Patterns

To search the dictionary effectively, we convert word patterns to a string so that they remain the same regardless of whether there is substitution or not. This allows us to perform efficient searches in the dictionary.

Caveats

When using dictionary attacks, there are certain caveats to consider:

  • Often resolves to several words
  • Longer words are better
  • Word may not be in the dictionary

Summary

In summary, here are some key points to remember:

  • You should not create your own cryptographic system without an experienced team of cryptanalysts exploring any flaws within the design.
  • To establish reliable and secure communication, never use an untested crypto algorithm.
  • While some attacks may not be possible on certain crypto algorithms, certain attacks, like letter frequency attacks on substitution ciphers, are inevitable.
  • Modular arithmetic is an important concept for representing historical encryption schemes, such as the Caesar cipher.
  • Dictionary attacks can aid in deciphering a ciphertext produced from a substitution cipher faster.
