Week 5: Privilege Escalation

Privilege Escalation Overview

Privilege escalation is a critical aspect of ethical hacking, involving the elevation of user privileges to gain unauthorized access to systems or sensitive information. Understanding different techniques and methods is crucial for identifying and mitigating potential vulnerabilities.

Privilege escalation can occur at various levels, including local privilege escalation on a single system and vertical privilege escalation, where an attacker gains access to higher-level privileges.

Local Privilege Escalation

Local privilege escalation involves exploiting vulnerabilities to elevate privileges on a single system. Common techniques include:

  • Kernel Exploits: Exploiting vulnerabilities in the operating system's kernel to gain elevated privileges.
  • File Permission Manipulation: Modifying file permissions to gain access to restricted areas.
  • Executable Exploits: Exploiting vulnerabilities in executable files to execute code with higher privileges.
  • Service Misconfigurations: Identifying and exploiting misconfigurations in system services for privilege escalation.
  • Abusing SUID/GUID: Exploiting set-user-ID (SUID) or set-group-ID (GUID) binaries to execute commands with elevated privileges.

Vertical Privilege Escalation

Vertical privilege escalation involves gaining higher-level privileges within an organization's hierarchy. This can occur through:

  • Exploiting Weak User Authentication: Cracking or obtaining passwords to impersonate higher-privileged users.
  • Abusing Service Accounts: Exploiting vulnerabilities in service accounts or using compromised credentials for higher access.
  • Pass-the-Ticket Attacks: Utilizing Kerberos tickets to move laterally and escalate privileges.
  • Exploiting Group Policy Weaknesses: Identifying and abusing weaknesses in group policies for privilege escalation.
  • Targeting Active Directory: Exploiting vulnerabilities in Active Directory to gain administrative privileges.

Mitigation Strategies

Preventing privilege escalation is crucial for maintaining a secure environment. Implement the following strategies:

  • Regular Patching: Keep systems and software up-to-date to address known vulnerabilities.
  • Least Privilege Principle: Assign minimum necessary privileges to users and services to limit the impact of potential escalations.
  • Monitoring and Logging: Implement robust monitoring and logging systems to detect and respond to suspicious activities.
  • Access Controls: Enforce strong access controls, ensuring users have access only to the resources necessary for their roles.
  • User Training: Educate users about the risks of social engineering and the importance of strong passwords.
  • Privilege Escalation Testing: Regularly conduct ethical hacking tests to identify and address potential privilege escalation vulnerabilities.

Additional Information

Privilege Escalation (Summary)

  • What can I do? - Retrieve passwords/keys, access sensitive/confidential files, exploit vulnerable OS/software/applications, exploit incorrect file permissions/misconfigurations, network attacks.
  • Who am I? - User identity (whoami, id), user rights (sudo, permissions, groups), shell type (tty shell), breakout shell, running cmd as administrator in Windows XP.
  • What is my environment? - OS details, kernel version (e.g., dirtycow exploit in Ubuntu), environment details (env command in Linux), allowed programming languages and tools (e.g., gcc, python), file upload methods.
  • How do users use the system? - User activities (e.g., who else is logged on, history, profiles, cache), directory services enumeration, sensitive information discovery.
